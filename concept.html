Spatial Memory

Concept, mechanism, and interaction in condensed form. Full text and figures are in the Full Doc.

1Concept

Spatial Memory closes a four-step loop: a worker captures a judgment by speaking in place at low effort, the system anchors it, structures it in a Memory Graph, and surfaces its current state to the next person.

It carries encounter-class information: low value alone, tied to a physical thing, with no known reader when spoken. Today's channels price this out, so it is lost. Known-recipient, bounded messages, which are episode-class, stay on Slack.

Every note is one of two memory types:

TypeAnchored toOutlivesExample
Object memorya placeany project"0.6 nozzle is in" on the printer
Project memorya project, spanning placesnothing; dies with the project"joint fragile" follows the build to the test bench

Person and time are not types but queries over the graph: whose trail, addressed to whom, since when.

2Memory Graph

The structuring step is a small typed store, in effect an advanced filter over what a room accumulates. It does three jobs:

JobWhat it does
State, not loga newer note supersedes an older one, so a place shows its current state
Routinga note follows its project memory, surfacing at another place where the work continues
Audience and decayeach arriver hears only what is for them, and notes expire when they no longer apply

The value is its position in the loop, not the data model, which a plain table could hold. The hard part is building it from messy in-flow speech.

3Interaction

Six moments of use, one per turn of the loop:

MomentWhat the user does or sees
Speak in placeone sentence mid-task, no app, no recipient
Surface on approachthe place replays its current notes, including any routed here by your project
Triage on arrival"anything for me?" returns a few ordered items
Follow a trailreplay one person's stations in order to catch up
Supersedesay the update; the old note collapses to current state
Let goephemera expire; project notes die with the project
